## 🚀 4. 今回 Claude Code に依頼する新規要件 (Health OS 実装フェーズ)

直近のセッション（Antigravityとの対話）にて、以下の「高度な疲労回復理論」をアプリに統合し、単なる記録ツールから**「データドリブンな回復サポートツール」**へと進化させる方針が決定しています。これらが優先度の高い実装タスクです。

### ① 【朝のチェックインの高度化】
*   **VAS (Visual Analogue Scale) の導入:** 定型的な選択肢ではなく、スライダーUI等を用いてコンディション（疲労度、気分の良さなど）を直感的かつ連続的な数値で記録できるように変更。
*   **リカバリー・タグの記録:** 昨晩から今朝にかけて実施した回復行動を記録。
    *   **ベース理論**: 「7つの休息（物理・精神・感覚・創造・感情・社会・霊的）」および「DRAMMAモデル（Detachment, Relaxation, Autonomy, Mastery, Meaning, Affiliation）」。
    *   *要件*: これらをユーザーが選びやすい「タグ形式（例：[感覚オフ], [家族との対話] など）」のUIで実装。

### ② 【夜のチェックアウト (Evening Check-out) システムの新規実装】
*   **1日の振り返りUX:** 一日の終わりのパフォーマンス（集中力、イライラ度、充実感）を評価する機能。
*   **睡眠阻害因子のトラッキング:** 夕食の時間、カフェインの最終摂取時刻（ルール：13:40がデッドライン）など、翌日のコンディションに直結する行動を記録。

### ③ 【データドリブン分析と可視化 (Radar Charts & Reset Equations)】
*   **分析とダッシュボード:** 朝の回復タグと夜のパフォーマンスデータの相関関係を分析する機能の基盤作成。
*   **レーダーチャート:** 「どの種類の休息が取れているか／不足しているか」を視覚化。
*   **あなただけの方程式 (Reset Equations):** データが蓄積された際に、「感覚休息を取った翌日はパフォーマンスが高い」といったパーソナライズされた気づきやアドバイスを自動生成・提示するロジックの構築。
